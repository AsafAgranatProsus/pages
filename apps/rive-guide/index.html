CH 1

Foundations & the Core Paradigm

Shift from a timeline-only, frame-by-frame mindset (After Effects / Lottie) to an interactive, state-driven one.

What is Rive?

Lesson 1.1

Rive is "a new general-purpose graphics format for all software and UI — animation is just part of the puzzle." A custom GPU renderer keeps files small and playback smooth.

  • Binary .riv at runtime vs Lottie's text-parsed JSON
  • Interactive 2D runtime vs Spline's spatial 3D

CH 2

The State Machine & Event Listeners

Eliminate the designer-to-developer handoff by wiring interactive logic visually.

State Machine Overview

Lesson 2.1

A state machine is "a decision-maker that chooses the right state (and animation) based on what's happening." A state is usually a timeline (Idle, Pressed), or a blend, or a placeholder like Exit.

  • Entry, Any State, Exit nodes
  • Layers run concurrently — idle breathing + looping blink
  • Each artboard is self-contained: own artwork, animation, state machine

Inputs & Conditions

Lesson 2.2

The three fundamental inputs that drive transitions:

Boolean — true/false toggles (e.g. hovering)
Number — math operators, numeric progression
Trigger — one-off action flashes

No-Code Listeners

Lesson 2.3

Map pointer actions straight to vectors — no engineer intervention. Common early mistake: setting the user action but forgetting to set the Rive action (the value the listener writes).

  • Pointer Enter / Exit / Move / Click
  • Listeners update bindings, set inputs, fire events

Blend States (1D & Additive)

Lesson 2.4

The hardest part of transitions: the in-between. Smoothly drive a goal-meter vector when a bound numeric value sits unpredictably between 40% and 75%.

Tune duration (e.g. 350ms in, 200ms back) and interpolation (elastic / bounce / snappy ease) live, while the state machine plays.

CH 3

Vector Rigging & Mesh Deformations

Turn static shapes into modular, living skeletons for character and UI motion.

The Three Pillars of Rigging

Lesson 3.1
Bones — skeletal structures driving vector paths
Joysticks — 2D controls syncing X/Y to facial expressions or cursor tracking
Constraints — automated bounds on Translation, Scale, Rotation

Inverse Kinematics

Lesson 3.2

Forward (FK): manual parent-child bone rotation.

Inverse (IK): limbs auto-calculate joint angles to follow one strategic target group.

Raster Meshes & PSD Imports

Lesson 3.3

Import structured .psd files preserving layers. Apply vector vertex meshes over raster art to morph textures — ripple cloth, flow hair, flex skin.

Bone Weighting & Vertex Binding

Lesson 3.4

Paint influence maps onto individual mesh vertices so bends read as seamless, organic deformation rather than rigid pivots.

CH 4

Data Binding & Procedural Logic

Make almost everything in a Rive graphic dynamically updatable at runtime.

Data Binding Core Concepts

Lesson 4.1

A View Model is "a box that contains all the data for this artboard." Create one, add typed fields (boolean, number, string, color, image, list), bind them to transition conditions.

Rive doesn't fetch data — it reacts to it. When a bound value updates, the right transition fires; the rest of the machine keeps humming, never blocking on an API.

View Models, Enums & Solos

Lesson 4.2

Build scalable architectures: nest View Models inside each other, map visual variants via Enums, switch component visibility instantly via Solos.

Functions & Automation

Lesson 4.3

Procedural workflows inside the editor — Formula Converters and Range Mappers to scale structural changes as repeatable, multi-step routines.

The Rive AI Coding Agent

Lesson 4.4

Script complex physics or camera logic with prompt-driven assistance. The loop: Prompt → Test → Refine.
